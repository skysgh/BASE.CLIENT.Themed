Learn how to define schema-driven aggregates with the BREAD pattern
What is the Entity Schema DSL?

The Entity Schema DSL (Domain-Specific Language) allows you to define your domain entities in a declarative JSON format. This schema drives the entire UI rendering pipeline — from browse views with filters and sorting, to edit forms with validation, to detail views — all from a single source of truth.

The BREAD Pattern
B
Browse
List/search entities with filtering
R
Read
View entity details (readonly)
E
Edit
Update existing entities
A
Add
Create new entities
D
Delete
Remove entities (soft/hard)
